1.1 The Box Model

The Box Model: Every element interpreted by the browser is inside a box. The box model has four properties, the content area's size (width and height) and the element's padding border, and margin.


Content Area: Properties height & width

Borders: Note, this is the property and the values are below

  • style: none, dotted and solid. There are more to choose from
  • color: normal color format

Border Radius: You can modify the corners of an element's border box with the border-radius property. You can create a border that is a perfect circle by setting the value to 100%.

Padding: This is the space between the contents of a box and the borders of a box. Imagine a mattboard in a picture frame, with the frame being the boarder. The padding property is often used to expand the background color and make content look less cramped. You can also do padding-top....

Margins: This is the space directly outside of the box. This means other elements can't come within x amount of pixels of the elements border.

Auto: The margin property also lets you center content. syntax is margin: 0 auto;. This will center the an object within its parent element.

Margin Collapse: The larger of the two vertical margins sets the distance between adjacent elements. Horizontal items are unlike.

Min and Max Height and Width: This is for different display sizes.

Overflow: The overflow property controls what happens to content that spills, or overflows, outside its box. the overflow property is set on a parent element to instruct the child elements. It can be set to the following values:

  • hidden: Any content that overflows will be hidden.
  • scroll: A scrollbar will be added to the element's box.
  • visibleThe overflow content will be displayed outside of the containing element.

Resetting Defaults: All major web browsers have a default stylesheet they use in the absence of an external stylesheet. You can change this with the * selector and setting the margin and padding to 0.

Visibility: Elements can be hidden with the visibility property. Values: hidden & visible.


1.2 Changing the Box Model

Under the current box model, the border thickness and padding are added to the overall dimensions of the box. This can make it difficult to accurately size a box.

content-box: The box-sizing property controls the type of box model the browser should use when interpreting a web page. The default value of this property is content-box (Affected by padding and border thickness).

border-box: This box models setting avoids the dimensional issues that exist in the former box model you learned about. In this box model, the height and width of the box will remain fixed, with the border thickness and padding being included inside of the box, hence the overall dimesions of the box do not change. Therefore, a 200px width would be a 200px width with the contents of the width being auto adjusted, depending on the padding and broder dimensions. To implement it in the browser, use the universal selector (*).


2. Display and Positioning


The default flow of the HTML documents is from left to right, top to bottom.

Position: The position property can have the following values.

  • Relative: This allows us to move the box in a relative position to the static. to move the box around, we need to use the top, bottom, left and right properties in conjuction with the posision property. With the directions, the box will remiain in the static position.
  • Absolute: When an element's position is set to absolute, all other elements on the page will ignore the element and act like it is not present on the page. The element will be positioned relative to its closest positioned parent element.
  • Fixed:A fixed element will scroll with the rest of the document and will always be in view. Good for navigation bars.

z-index: The z-index property controls how far 'back' or forward an element should appear. It accepts integer values.

display: The display property dictates if an element can share the same horizontal space with other elements. The three values for the display property:

  • inline: elements takes up as little space as possible, and they cannot have manually-adjsuted width or height.
  • block: elements take up the width of their container and can have manually-adjusted heights.
  • inline-block: elements can have set width and height, but they can also appear next to ech other and do not take up thier entire container width.

Float: If you're simply interested in moving an element as far left or as far right as possible, you can use the float property. The values are left & right.

clear: property specifies how elements should behave when they bump into each other on the page. It can take on the following values:

  • left: The left side of the element will not touch any other element within the same containing element.
  • right: The right side of the element will not touch any other element within the same containing element.
  • both: neither side of the element will touch any other element within the same containing element.
  • none: the element can touch either side.

font family: When setting typefaces on a web page, the font specified in a stylesheet must be installed on a user's computer in order for that font to display when a user visit the web page. When the name of a typeface consists of more than one word, it must be enclosed in double quotes. It may be useful to add fallback fonts which come from a pre-installed family. Google fonts is a good directory which contain thousands of open-source fonts. Link the font inbetween the head tags using hte link tag, with attributes href, type and rel.

  • Serif: Extra detail
  • Sans Serif: no extra detail

font weight: Has the properties bold and normal. This property can also be assigned a value from 100 - 900 in multiples of 100, where 400 indicates default text, anything higher is bold, and anything lower is light.

font style: This property has italic and normal values.

letter-spacingThis technique is called tracking and can be useful on uppercat text. Increase the space between words in em units.

text-transformation : Has values uppercase and lowercase.

text-align : Has values left,center and right.

line-height: This property can set how the spacing between each line which can improve legibility. It can take values such as, a unitless number, which is a ration of the font size, and a number specified by a unit. Generally, the unitless ratio value is the preferred emthod since it is responsive.

@font-face : This is a way to import fonts directly into stylesheets with the @font-face property. To do this, load the font url onto the browser and you willl be redirected a the page of css files. You can also modify @font-face to use local font files. Here you will have url and format value.


5.1 Responsive Design

Relative Measurements: Responsive design refers to the ability of a website to resize and reogainze its content based on the size of other content on the website and the size of the display. The unit px is a hard code whem units em and rem are relative measurements.

  • em: This unit is based of the parent element font size being used.
  • rem: This unis is based of the root elements font size in the <html> tag. One advantage of using rems is that all elements are compared to the same font size value.

Percentages: Height & Width: To size non-text HTML elements relative ot their parent elements on the page you can us percentages. Percentages will size relative to the dimensions of the parent element (also known as the contianer).
Note: for the boxmodel 100% should only be used when content will not have padding, border or magin.

Percentages: Padding & Margin: When setting these two by percentages, they are calculated based ony the width of the parent element.

Width: Minimum & Maximum: This will limit how wide an element becomes. This can be usefull to prevent a text being too compressed or too spreadout. Use the units of pixels to ensure hard limits are being used on the dimensions.

Height: Minimum & Maximum: Same application as above.

5.2 Media Queries

Here we will focus on how to deal with different screen sizes

Media Queries: With media queiries, CSS can detect the size of the current screen and apply different CSS styles depending on the width of the screen. You will need to write the following syntax in your css file: @media only screen and (max-width: 480){"""}. This is just an exmaple.

Range: You can specify a range by declaring a min-width/height and max-width/height.

Dots Per Inch (DPI): Another feature we can set target to is screen resolution. dpi is a unit of measurement. If a resoultion is met, we can use CSS to display high resolution media.

And Operator: This allows us to chain multiple media features of the sme type in one media query.

Comma Sepearated List: If only one of multiple media features in a media query must be met, media features can be separated in a comma separated list.

Breakpoints: The points at which media queires are set are called breakpoints. Breakpoints are the screen sizes at which your web page does not appear properly. Rather than set breakpoints based on specific devices, the best practice is to resize your browser to view where the website naturally breaks based on its content.



6. Flexbox

Flexbox is not meant to lay out entire pages, but it is useful for positioning elements, whether in individually or in groups. There are two important components to a flexbox layout: flex containers and flex items. A container is an elemnt on a page that contains flex items.

display: flex: For an element to become a flex continaer, its display property must be set to flex. Now the children of the flex container will become flex items. A div with declaration display: flex, will remain block level.

display: inline-flex: This is if we want multiple flex containers to display inline with each other.

justify-content (property, container): When we change the display to either flex or inline-flex, all of the flex items then move toward the upper left corner of the parent container (default behaviour). justify-content positions the items from left to right. There are five values for this property:

  • flex-start
  • flex-end
  • center
  • space-around
  • space-between

align-items(container): This property flex's items vertically. There are five values for this property:

  • flex-start
  • flex-end
  • center
  • baselineThe bottom of the content of all items will be aligned with each other.
  • stretch: This will stretch the items from top to bottom of the container. Elements with specific heights and restriction will not stretch.

flex-grow (items): All flex items will shrink if the container is too small, however if the container is larger than necessary, the items will not stretch by default. The flex-grow property allows us to specify if items should grow to fill a container and also which items should grow proporitionally more or less than others.

flex-shrink (items): Just like flex-grow, this property can be used to specify which elements will shrink and in what proportions. The value 0 will indicate nothing, whilst 1, 2 and 3 will shrink faster and smaller via order of magnitude.

flex-basis (items): This property allows us to specify the width of an item before it stretches or shrinks. This values of the property is in units.

flex (items): The flex property combines all of the three propertys above into one. The values will be written in the same manner, only separated by a space.

flex-wrap (container): With this property, our flex items will move to the next line when necessary, instead of shrinking to fit in the container. This property can accept three values:

  • wrap: Child elements of a flex container that don't fit into a row will move down to the next line.
  • wrap-reverse: The same but the order of rows are reverse.
  • nowrap: This is the default value.

align-content(container): If a flex container has multiple rows of content, we can use this property to space the rows from top to bottom. This is different to align-items, where the later is used for single rows. This property has six values:

  • flex-start
  • flex-end
  • center
  • space-between
  • space-around
  • stretch

flex-direction (container):This property allows us to swithc the major axis(horizontal) and cross axis(vertical). Major axis is used to position flex items with justify-content, flex-wrap, flex-grow and flex-shrink. The cross axis edits with algin-items and align-content. This property can accept four values:

  • row
  • row-reverse
  • column
  • column-reverse

flex-flow (containers): This property is used to declare both flex-wrap and flex-direction properties in one line. The first value is for the flex flow and the second is for the flex directions property.

Nested containers:Positioning flex containers inside of one another.
